# Trademarks and Branding

This file is a **policy statement**, not a software license.

## Code vs. brand
- The **code** is governed by `LICENSE` (PolyForm Noncommercial 1.0.0).
- Names, logos, and branding are governed separately by trademark and unfair-competition laws.

## Forks are allowed; confusing branding is not
You may fork the repository as permitted by the license, but you may not use the original project’s name/logo/branding in a way that:
- implies your fork is official, endorsed, or maintained by the original author, or
- makes your distribution look like an “official” build.

## Allowed uses (examples)
- Referencing the project name to describe origin (e.g. “forked from stable-diffusion-webui-codex”).
- Linking back to the original repository.

## Not allowed uses (examples)
- Using the same name/logo to market a competing hosted service.
- Using the branding to sell a clone “as if it were official”.

## Recommended fork hygiene
- Rename your fork/project.
- Remove/replace logos and “official” wording.
- Add a clear disclaimer that your fork is not affiliated with the original project.
